# Table Editor

[English](README.md)

Настольное Windows-приложение для создания, редактирования и сохранения табличных данных с GPU-ускоренной сортировкой столбцов и вычислением формул через CUDA.

---

## Возможности

- Создание таблиц с произвольным числом строк и столбцов
- Открытие и сохранение таблиц в формате CSV
- Редактирование ячеек прямо в таблице
- Добавление и удаление строк и столбцов
- **GPU-ускоренная сортировка столбцов** с помощью CUDA-ядра на основе сортировки выбором
- **GPU-ускоренное вычисление формул** — введите выражение, начинающееся с `=`, для вычисления арифметического результата
- **Ссылки на ячейки в формулах** — формат `{строка}A{столбец}` (например, `=1A2+3A4`)
- Обнаружение циклических зависимостей (`#CIRC!`)
- Пересчёт всех формул кнопкой **⟳ Recalc**
- Горячие клавиши: `Ctrl+N` (создать), `Ctrl+O` (открыть), `Ctrl+S` (сохранить)

---

## Синтаксис формул

Формулы начинаются с `=`. Ссылки на ячейки записываются как `{строка}A{столбец}`, где строка и столбец — целые числа, нумерация с 1.

| Ввод | Смысл |
|------|-------|
| `=5+3` | Вычисляет `5 + 3` на GPU |
| `=1A2+1A3` | Складывает значения: строка 1 столбец 2 + строка 1 столбец 3 |
| `=2A1*3A1` | Умножает значения: строка 2 столбец 1 × строка 3 столбец 1 |

**Поддерживаемые операторы:** `+`, `-`, `*`, `/` (вычисляются слева направо, без приоритетов).

**Коды ошибок:**

| Код | Причина |
|-----|---------|
| `#CIRC!` | Обнаружена циклическая ссылка |
| `#REF!` | Адрес ячейки выходит за пределы таблицы |
| `#TYPE!` | Ссылочная ячейка содержит нечисловую строку |
| `#ERR: …` | Ошибка вычисления на GPU |

## Требования

- Windows 10/11 (x64)
- [.NET 6.0 Desktop Runtime](https://dotnet.microsoft.com/en-us/download/dotnet/6.0)
- Видеокарта NVIDIA с поддержкой CUDA (Compute Capability 5.0+)
- Библиотеки [CUDA Toolkit 10.x](https://developer.nvidia.com/cuda-toolkit-archive)

---

## Сборка

1. Установите [.NET 6 SDK](https://dotnet.microsoft.com/en-us/download/dotnet/6.0) и Visual Studio 2022.
2. Установите CUDA Toolkit 10.x и убедитесь, что `nvcc` доступен в `PATH`.
3. Откройте `table_editor.sln` в Visual Studio и выполните сборку, либо запустите:

```
dotnet build table_editor.sln
```

Скомпилированные PTX-файлы ядер (`selectionsort.ptx`, `evaluate.ptx`) уже собраны и автоматически копируются в выходную директорию. Для пересборки из исходников:

```bash
nvcc -ptx selectionsort.cu -o selectionsort.ptx
nvcc -ptx evaluate.cu -o evaluate.ptx
```

## Запуск тестов

```
dotnet test table_editor.Tests
```

Тестовое покрытие: `Cell`, `Row`, `TableModel`, `FormulaParser` — вся чистая логика, не зависящая от GPU.

---

## Формат CSV

Таблицы хранятся в CSV-файлах с разделителем `;`. Каждая строка заканчивается точкой с запятой и переводом строки. Первый столбец содержит номер строки (в редакторе он только для чтения).

```
1;Alice;30;Engineer;
2;Bob;25;Designer;
```

## Структура проекта

```
table_editor/
├── classes/
│   ├── Cell.cs              — Ячейка: определение типа, отображаемое значение
│   ├── CellType.cs          — Перечисление: String / Integer / Float / Formula
│   ├── Row.cs               — Наблюдаемая строка (ObservableCollection<Cell>)
│   ├── TableModel.cs        — Чистая модель данных (без зависимости от WPF)
│   ├── CudaDataTable.cs     — WPF-обёртка: столбцы + вычисление формул
│   ├── CudaSorter.cs        — GPU-сортировка столбца (IColumnSorter)
│   ├── CudaEvaluator.cs     — GPU-вычислитель формул (IFormulaEvaluator)
│   ├── FormulaParser.cs     — Парсер ссылок на ячейки (чистый, тестируемый)
│   ├── IColumnSorter.cs     — Интерфейс сортировщика
│   └── IFormulaEvaluator.cs — Интерфейс вычислителя
├── windows/
│   └── NewTableWindow.xaml  — Диалог создания таблицы
├── table_editor.Tests/      — Тестовый проект (xUnit)
├── *.cu / *.ptx             — Исходники и бинарные CUDA-ядра
└── icons/                   — Иконки панели инструментов
```
